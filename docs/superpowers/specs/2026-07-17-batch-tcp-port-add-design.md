# 批量添加 TCP 端口设计

## 目标

在现有 TCP 端口工具栏中增加批量添加入口，允许用户一次输入多个非连续 TCP 端口，并为每个有效的新端口创建完整的端口会话页面、命令表、日志区域和实时统计栏。

## 交互设计

- 在 TCP Ports 工具栏增加 `Batch Add` 按钮。
- 点击后弹出文本输入框。
- 输入支持逗号、空格、分号和换行分隔，例如：

  ```text
  10160,10165 10200;10300
  ```

- 原有 `+ Add Port` 单个添加按钮保持不变。

## 校验与结果

- 端口必须是十进制整数，范围为 `1–65535`。
- 重复输入只处理一次。
- 已存在于当前 TCP 端口列表的端口跳过。
- 非法输入不影响其他合法端口处理。
- 添加结束后提示成功添加、重复跳过和非法输入的数量。
- 空输入不创建端口并给出提示。

## 实现边界

- 批量入口只支持 TCP，不改动串口功能。
- 每个新端口统一调用现有 `addTcpPort(quint16)`，复用已有的会话、命令表、日志、统计和连接初始化逻辑。
- 不改变连接、发送、Interval、Timeout、统计或日志文件命名规则。
- 使用现有 `m_tcpSessions` 去重，避免重复创建相同端口。

## 测试要求

至少验证以下场景：

1. 输入多个合法的非连续端口，全部创建成功。
2. 输入重复端口和已存在端口，均不会重复创建。
3. 混合合法端口、非法字符串和越界数字，合法项仍成功创建。
4. 空输入不创建端口并提示用户。
5. 添加 `1` 和 `65535` 边界端口成功，`0`、负数和大于 `65535` 的端口被拒绝。
6. 编译通过，现有协议测试继续通过。
